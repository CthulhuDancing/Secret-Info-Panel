# Workflow

1. Start new revisions from `live/secret_info_panel_live.naiscript`.
2. Test and validate revision work before promotion.
3. Promote a tested revision into `live/secret_info_panel_live.naiscript` when it becomes the authoritative working script.
4. Store versioned historical scripts in `archive/code/`.
5. Store fork or proposal artifacts in `archive/forks/`.
6. Keep current working docs in `docs/`.
7. Move superseded docs into `archive/docs/`.
8. Store test exports, logs, and related artifacts under `data/`.
